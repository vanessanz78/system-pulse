# Real-World Learnings

System Pulse evolves through observation.

This folder records real founder and user observations that should improve PulseCore over time. These are not treated as ordinary bugs. Each note captures what happened, what PulseCore should learn, what is implemented now, and what remains future research.

Use this format:

- Observation: what happened in real use.
- Signals: what was measured or noticed.
- Product learning: what System Pulse should understand next.
- PulseCore response: how reasoning should change.
- Implementation state: current, partial, future, or researched only.

The goal is to grow a durable library of real-world computer behaviour that makes System Pulse more useful without drifting into invented assumptions.
